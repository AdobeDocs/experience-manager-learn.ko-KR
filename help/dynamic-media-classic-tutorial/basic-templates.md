# 기본 템플릿 소개 {#basic-templates}

Dynamic Media Classic에서 템플릿은 템플릿이 게시된 후 URL을 통해 동적으로 변경할 수 있는 문서입니다. Dynamic Media Classic에서는 이미지 서버에서 호출되는 이미지 기반 템플릿과 이미지 및 렌더링된 텍스트로 구성된 기본 템플릿을 제공합니다.

템플릿의 가장 강력한 측면 중 하나는 직접 통합 지점을 사용하여 데이터베이스에 연결할 수 있다는 것입니다. 따라서 이미지를 제공하고 크기를 조정할 수 있을 뿐만 아니라, 데이터베이스를 쿼리하여 새 항목 또는 판매 항목을 찾고 이미지에 오버레이로 나타나도록 할 수 있습니다. 항목에 대한 설명을 요청하고 선택한 글꼴과 레이아웃에 레이블로 표시되도록 할 수 있습니다. 가능성은 무한합니다.

기본 템플릿은 단순에서 복잡성에 이르기까지 다양한 방법으로 구현할 수 있습니다. 예:

- 기본 머천다이징. 해당 제품에 무료 배송이 가능한 경우 &quot;무료 배송&quot;과 같은 레이블을 사용합니다. 이러한 레이블은 Photoshop의 상품 팀이 설정하며 웹은 논리를 사용하여 이미지에 적용할 시기를 결정합니다.
- 고급 머천다이징. 각 템플릿에는 여러 변수가 있으며 동시에 두 개 이상의 선택 사항을 표시할 수 있습니다. 데이터베이스, 재고 및 비즈니스 규칙을 사용하여 &quot;Clearance&quot; 또는 &quot;Sold Out&quot;에서 제품을 &quot;Just In&quot;으로 표시할 시기를 결정합니다. 또한 제품 뒷면의 투명도를 사용하여 다른 방에서와 같이 다른 배경에서 표시할 수도 있습니다. 동일한 템플릿 및/또는 자산을 제품 세부 사항 페이지에서 다시 지정하여 다른 배경에서 동일한 제품의 더 크거나 확장 가능한 버전을 표시할 수 있습니다.

Dynamic Media Classic에서는 이러한 템플릿 기반 애플리케이션의 시각적 부분만 제공한다는 것을 이해하는 것이 중요합니다. Dynamic Media Classic 회사나 통합 파트너는 애플리케이션을 만들기 위해 비즈니스 규칙, 데이터베이스 및 개발 기술을 제공해야 합니다. 기본 제공 템플릿 응용 프로그램이 없습니다. 디자이너는 Dynamic Media Classic에서 템플릿을 설정하고 개발자는 URL 호출을 사용하여 템플릿의 변수를 변경합니다.

자습서의 이 섹션을 마치면 다음을 수행하는 방법을 알 수 있습니다.

- Photoshop PSD를 Dynamic Media Classic에 업로드하여 템플릿 기반으로 사용할 수 있습니다.
- 이미지 레이어로 구성된 간단한 머천다이징 기본 템플릿을 만듭니다.
- 텍스트 레이어를 추가하고 매개 변수를 사용하여 변수로 만듭니다.
- 템플릿 URL을 구성하고 웹 브라우저를 통해 이미지를 동적으로 조작할 수 있습니다.

>[!NOTE]
>
>이 장의 모든 URL은 예시적인 용도로만 사용됩니다. 라이브 링크가 아닙니다.

## 기본 템플릿 개요

기본 템플릿의 정의(또는 &quot;템플릿&quot;만 해당)는 URL 주소 지정 가능한 레이어 이미지입니다. 최종 결과는 이미지이지만 URL로 변경할 수 있습니다. Dynamic Media Classic의 모든 P-TIFF 자산 조합인 사진, 텍스트 또는 그래픽으로 구성될 수 있습니다.

템플릿은 유사한 워크플로우 및 유사한 기능을 가지고 있으므로 Photoshop PSD 파일과 가장 유사합니다.

- 두 층은 모두 아세테이트 적층판과 같은 층으로 구성되어 있다. 부분적으로 투명한 이미지를 합성하고 레이어의 투명 영역을 통해 아래 레이어에 확인할 수 있습니다.
- 레이어를 이동 및 회전하여 컨텐츠 위치를 변경할 수 있으며 불투명도 및 혼합 모드를 변경하여 컨텐츠를 부분적으로 투명하게 만들 수 있습니다.
- 텍스트 기반 레이어를 만들 수 있습니다. 이미지 서버는 Photoshop 및 Illustrator과 동일한 텍스트 엔진을 사용하므로 품질이 매우 높을 수 있습니다.
- 각 레이어에 간단한 레이어 스타일을 적용하여 그림자 또는 광선과 같은 특수 효과를 만들 수 있습니다.

그러나 Photoshop PSD와 달리 레이어는 이미지 서버의 URL을 통해 완전히 동적이고 제어할 수 있습니다.

- 모든 템플릿 속성에 변수를 추가할 수 있으므로 컴포지션을 즉시 쉽게 변경할 수 있습니다.
- 매개 변수라는 변수를 사용하면 변경할 템플릿의 부분만 표시할 수 있습니다.

Photoshop에서와 마찬가지로 모든 레이어를 단일 파일에 넣고 표시 및 숨기는 대신 가변할 각 레이어에 대한 자리 표시자만 추가해야 합니다(원하는 경우 이러한 작업도 수행할 수 있음).

자리 표시자를 사용하여 레이어의 컨텐츠를 게시된 다른 자산으로 동적으로 교체할 수 있으며, 레이어의 크기 및 회전 등의 동일한 속성을 자동으로 가져옵니다.

기본 템플릿은 일반적으로 Photoshop에서 디자인되지만 URL을 통해 배포되므로 템플릿 프로젝트에는 디자인과 기술 기술이 모두 혼합되어 있어야 합니다. 일반적으로 크리에이티브 템플릿 작업을 수행하는 사람은 Photoshop 디자이너이며 템플릿을 구현하는 사람은 웹 개발자라고 가정합니다. 템플릿을 성공적으로 사용하려면 크리에이티브 및 개발 팀이 함께 협력해야 합니다.

템플릿 프로젝트는 비즈니스 규칙과 애플리케이션의 요구 사항에 따라 비교적 간단하거나 매우 복잡할 수 있습니다. 기본 템플릿은 이미지 서버에서 호출되지만 Dynamic Media Classic 환경의 유연성으로 인해 다른 템플릿 내에 템플릿을 중첩할 수 있으므로 일반적으로 이름이 지정된 변수로 연결할 수 있는 매우 복잡한 이미지를 만들 수 있습니다.

- [템플릿 기본 사항](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/template-basics/quick-start-template-basics.html)에 대해 자세히 알아보십시오.
- [기본 템플릿을 만드는 방법을 알아봅니다](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/template-basics/creating-template.html#creating_a_template).

## 기본 템플릿 만들기

기본 템플릿으로 작업할 때는 일반적으로 아래 다이어그램의 워크플로우 단계를 따릅니다. 점선으로 표시된 단계는 동적 텍스트 레이어를 사용하는 경우 선택 사항이며 아래 지침에 &quot;텍스트 워크플로우&quot;로 표시됩니다. 텍스트를 사용하지 않는 경우에는 기본 경로만 따르십시오.

![이미지](assets/basic-templates/basic-templates-1.png)

_기본 템플릿 워크플로우입니다._

1. 자산을 디자인하고 만듭니다. 대부분의 사용자는 Adobe Photoshop에서 이 작업을 수행합니다. 필요한 정확한 크기로 자산을 디자인합니다. 축소판 페이지의 200픽셀 이미지인 경우 200픽셀로 디자인합니다. 확대해야 한다면 약 2000픽셀 크기로 디자인하세요. Photoshop(및/또는 비트맵으로 저장된 Illustrator)을 사용하여 자산을 만들고 Dynamic Media Classic을 사용하여 부품을 함께 조합하고 레이어를 관리하며 변수를 추가합니다.
2. 그래픽 자산을 디자인한 후 Dynamic Media Classic에 업로드합니다. PSD에서 개별 자산을 업로드하는 대신 업로드 시 **레이어 유지** 옵션을 사용하여 전체 레이어 PSD 파일을 업로드하고 Dynamic Media Classic에서 레이어별 파일을 만드는 것이 좋습니다(자세한 내용은 아래 참조). _텍스트 워크플로우: 다이내믹 텍스트를 만드는 경우 글꼴도 업로드합니다. 동적 텍스트는 변수를 사용하며 URL을 통해 제어됩니다. 텍스트가 정적이거나 변경되지 않는 몇 가지 짧은 구만 있는 경우, 예를 들어 X가 변수 번호인 &quot;X% Off&quot;가 아닌 &quot;New&quot; 또는 &quot;Sale&quot;이라는 태그가 있는 경우 Photoshop에서 텍스트를 미리 렌더링하고 래스터화된 레이어로 이미지를 업로드하는 것이 좋습니다. 그러면 더 쉽고 원하는 대로 텍스트 스타일을 지정할 수 있습니다._
3. 빌드 메뉴의 템플릿 기본 사항 편집기를 사용하여 Dynamic Media Classic에서 템플릿을 작성하고 이미지 레이어를 추가합니다. 텍스트 워크플로우: 동일한 편집기에서 텍스트 레이어를 만듭니다. 이 단계는 Dynamic Media Classic에서 수동으로 템플릿을 작성할 때 필요합니다. 디자인과 일치하는 캔버스 크기를 선택하고, 이미지를 캔버스로 끌어다 놓고, 레이어 속성(크기, 회전, 불투명도 등)을 설정합니다. 템플릿에 모든 가능한 레이어를 배치하지 않고 이미지 레이어당 하나의 자리 표시자만 배치합니다. _텍스트 워크플로우: Photoshop에서 텍스트 레이어를 만드는 것과 유사한 [텍스트 도구]를 사용하여 텍스트 레이어를 만듭니다. Photoshop 문자 도구에서 사용할 수 있는 것과 동일한 옵션을 사용하여 글꼴을 선택하고 스타일을 지정할 수 있습니다._ 또 다른 워크플로우는 PSD를 업로드하고 Dynamic Media Classic에서 &quot;무료&quot; 템플릿을 생성하도록 하며 텍스트 레이어를 다시 만들 수도 있습니다. 이 내용은 나중에 자세히 설명합니다.
4. 레이어가 만들어지면 레이어의 소스( 이미지 자체)를 비롯하여 URL을 통해 제어할 레이어의 속성에 매개 변수(변수)를 추가합니다. _텍스트 워크플로우: 텍스트 레이어에 매개 변수를 추가하여 텍스트 내용과 레이어 자체의 크기 및 위치뿐만 아니라 글꼴 색상, 글꼴 크기, 가로 추적 등의 모든 서식 옵션을 제어할 수도 있습니다._
5. 템플릿 크기와 일치하는 이미지 사전 설정을 만듭니다. 템플릿 크기가 항상 1:1 크기로 호출되고 템플릿에 맞게 크기가 조정된 큰 이미지 레이어에 선명하게 하기를 추가하려면 이 작업을 수행하는 것이 좋습니다. 확대/축소할 템플릿을 만드는 경우 이 단계는 필요하지 않습니다.
6. Dynamic Media Classic 미리 보기에서 URL을 게시하고 복사한 다음 브라우저에서 테스트합니다.

## Dynamic Media Classic에 템플릿 자산 준비 및 업로드

Dynamic Media Classic에 템플릿 자산을 업로드하기 전에 몇 가지 준비 단계를 완료해야 합니다.

### 업로드할 PSD 준비

Photoshop 파일을 Dynamic Media Classic에 업로드하기 전에 Photoshop의 레이어를 간소화하여 작업하기 쉽게 하고 이미지 서버와 가장 잘 호환되도록 하십시오. PSD 파일은 Dynamic Media Classic에서 인식하지 못하는 많은 요소로 구성되며 관리하기가 어려운 작은 조각도 만들 수 있습니다. 나중에 원본을 편집해야 하는 경우 마스터 PSD의 백업을 저장해야 합니다. 마스터가 아니라 간소화된 복사본을 업로드합니다.

![이미지](assets/basic-templates/basic-templates-2.jpg)

1. 함께 설정/해제해야 하는 관련 레이어를 단일 레이어로 병합/병합하여 레이어 구조를 단순화합니다. 예를 들어 레이블 &quot;NEW&quot;와 파란색 배너가 단일 레이어로 병합되므로 한 번의 클릭으로 표시하거나 숨길 수 있습니다.
   ![이미지](assets/basic-templates/basic-templates-3.jpg)
2. 일부 레이어 유형 및 레이어 효과는 Dynamic Media Classic 또는 Image Server에서 지원되지 않으므로 업로드하기 전에 래스터화해야 합니다. 그렇지 않으면 효과가 무시되거나 레이어가 삭제될 수 있습니다. 레이어를 래스터화하는 것은 편집할 수 없을 때 로 변환하는 것을 의미합니다. 레이어 효과 또는 텍스트 레이어를 래스터화하려면 빈 레이어를 만들고 **레이어 > 레이어 병합** 또는 CTRL + E/CMD + E를 사용하여 모두 선택하고 병합합니다.

   - Dynamic Media Classic은 레이어를 그룹화하거나 연결할 수 없습니다. 그룹 또는 연결된 세트의 모든 레이어는 더 이상 그룹화되거나 연결되지 않은 별도의 레이어로 변환됩니다.
   - 레이어 마스크는 업로드 시 투명도로 변환됩니다.
   - 조정 레이어는 지원되지 않으며 삭제됩니다.
   - [단색 레이어]와 같은 칠 레이어는 래스터화됩니다.
   - 스마트 오브젝트 레이어와 벡터 레이어는 업로드 시 일반 이미지로 래스터화되며 스마트 필터가 적용되고 래스터화됩니다.
   - 텍스트 추출 옵션을 사용하지 않으면 텍스트 레이어도 래스터화됩니다. 자세한 내용은 아래 를 참조하십시오.
   - 대부분의 레이어 효과는 무시되고 몇 가지 혼합 모드만 지원됩니다. 확실하지 않은 경우 Dynamic Media Classic에서 간단한 효과(예: 내부 또는 그림자, 내부 또는 외부 광선)를 추가하거나 빈 레이어를 사용하여 Photoshop에서 효과를 병합하고 래스터화합니다.

### 글꼴을 사용한 작업

또한 동적 텍스트를 생성해야 하는 경우 글꼴을 업로드하고 게시합니다. Dynamic Media Classic에 포함된 유일한 글꼴은 Arial입니다.

각 회사는 웹 상에서 글꼴을 사용할 수 있는 라이센스를 얻는 것이 책임입니다. 단순히 컴퓨터에 글꼴이 설치되어 있다고 해서 사용자에게 웹에서 상업적으로 사용할 수 있는 권한이 부여되지 않으며, 허가없이 사용되면 글꼴 게시자의 법적 조치를 취할 수 있습니다. 또한 라이센스 용어는 다양하며 예를 들어 인쇄와 화면 표시에 별도의 라이센스가 필요할 수 있습니다.

Dynamic Media Classic은 표준 OpenType(OTF), TrueType(TTF) 및 Type 1 Postscript 글꼴을 지원합니다. Mac 전용 슈트케이스 글꼴, 유형 컬렉션 파일, Windows 시스템 글꼴과 전용 시스템 글꼴(음각 또는 자수 시스템에서 사용하는 글꼴 등)은 모두 지원되지 않으므로 표준 글꼴 형식 중 하나로 변환하거나 Dynamic Media Classic 및 Image Server에서 사용할 유사한 글꼴을 대체해야 합니다.

다른 자산과 마찬가지로 Dynamic Media Classic에 글꼴이 업로드되면 Image Server에도 게시되어야 합니다. 매우 일반적인 템플릿 오류는 글꼴을 게시하는 것을 잊어버리는 것입니다. 그러면 이미지 오류가 발생합니다. 이미지 서버는 다른 글꼴을 대신 대체하지 않습니다. 또한 업로드 시 **텍스트 추출** 옵션을 사용하려면 해당 글꼴을 사용하는 PSD를 업로드하기 전에 글꼴 파일을 업로드해야 합니다. **텍스트 추출** 기능은 텍스트를 편집 가능한 텍스트 레이어로 다시 만들어 Dynamic Media Classic 템플릿 내에 배치하려고 합니다. 다음 주제인 PSD 옵션에서 설명합니다.

글꼴에는 외부 파일 이름과 종종 다른 여러 내부 이름이 있습니다. Dynamic Media Classic에서 해당 자산에 대한 세부 사항 페이지에서 서로 다른 모든 이름을 볼 수 있습니다. 다음은 Dynamic Media Classic의 메타데이터 탭 아래에 나열된 Adobe Caslon Pro Semibold 글꼴의 이름입니다.

![이미지](assets/basic-templates/basic-templates-4.jpg)

_Dynamic Media Classic에서 글꼴에 대한 세부 정보 페이지의 메타데이터 탭._

Dynamic Media Classic에서는 이 글꼴의 파일 이름(ACaslonPro-Semibold)을 해당 자산 ID로 사용하지만 템플릿에 사용되는 이름이 아닙니다. 서식 파일은 맨 아래에 나열된 서식 있는 텍스트 형식(RTF) 이름을 사용합니다. RTF는 Image Server 텍스트 엔진의 기본 &quot;언어&quot;입니다.

URL을 통해 글꼴을 변경해야 하는 경우, 글꼴의 RTF 이름(자산 ID가 아님)을 호출해야 합니다. 그렇지 않으면 오류가 발생합니다. 이 경우 이 글꼴의 적절한 이름은 &quot;Adobe Caslon Pro&quot;입니다. 아래 RTF 및 텍스트 매개 변수에 글꼴과 RTF에 대해 자세히 설명합니다.

Windows 및 Mac 시스템에서 발견되는 가장 일반적인 글꼴 파일 형식은 OpenType 및 TrueType입니다. OpenType에는 .OTF 확장명이 있지만 TrueType은 .TTF입니다. Dynamic Media Classic에서는 두 형식 모두 잘 작동합니다.

### PSD를 업로드할 때 옵션 선택

템플릿을 만들려면 Photoshop 파일(PSD)을 업로드할 필요가 없습니다. 템플릿은 Dynamic Media Classic의 모든 이미지 자산으로 작성할 수 있습니다. 그러나 PSD를 업로드하면 일반적으로 이러한 자산이 레이어 PSD에 이미 있으므로 보다 쉽게 작성할 수 있습니다. 또한 레이어 PSD를 업로드할 때 Dynamic Media Classic에서 자동으로 템플릿을 생성합니다.

- **레이어 유지** 가장 중요한 선택이다. 이렇게 하면 Dynamic Media Classic에서 Photoshop 레이어당 하나의 이미지 자산을 만들 수 있습니다. 선택하지 않으면 다른 모든 옵션이 비활성화되고 PSD가 단일 이미지로 병합됩니다.
- **** **CreateTemplate.** 이 옵션은 생성된 다양한 레이어를 가져와 다시 결합하여 템플릿을 자동으로 만듭니다. 자동 생성된 템플릿을 사용하는 것의 단점은 Dynamic Media Classic이 모든 레이어를 하나의 파일에 배치한다는 것이지만 레이어당 하나의 자리 표시자만 있으면 됩니다. 추가 레이어를 삭제하기 쉽지만 레이어가 많으면 레이어를 다시 만드는 것이 더 빠릅니다. 새 템플릿의 이름을 반드시 바꾸십시오. 그렇지 않으면 다음에 동일한 PSD를 다시 업로드할 때 덮어쓰여집니다.
- **텍스트 추출.** 업로드한 글꼴을 사용하여 PSD의 텍스트 레이어를 템플릿의 텍스트 레이어로 다시 만듭니다. 텍스트가 Photoshop의 경로에 있고 템플릿에서 해당 경로를 유지하려는 경우 이 단계가 필요합니다. 이 기능을 사용하려면 업로드 시 생성된 템플릿에서만 추출된 텍스트를 만들 수 있으므로 **템플릿 만들기** 옵션을 사용해야 합니다.
- **레이어를 배경 크기로 확장합니다.** 이 설정은 각 레이어를 전체 PSD 캔버스와 동일한 크기로 만듭니다. 이 기능은 항상 고정된 위치에 있는 레이어에 매우 유용합니다. 그렇지 않은 경우 이미지를 동일한 레이어로 교체해야 할 수 있습니다.
- **레이어 이름 지정.** 이것은 Dynamic Media Classic에서 레이어별로 생성된 각 자산의 이름을 지정하는 방법을 알려줍니다. **Photoshop** **및 Layer** **이름** 또는 Photoshop 및 **레이어** **숫자**&#x200B;를 권장합니다. 두 옵션 모두 PSD 이름을 이름의 첫 부분으로 사용하고 끝에 레이어 이름이나 번호를 추가합니다. 예를 들어, &quot;shirt.psd&quot;라는 PSD가 있고 &quot;front&quot; &quot;feely&quot; 및 &quot;collar&quot;라는 레이어가 있는 경우, **Photoshop 및** Layer **Name** 옵션을 사용하여 업로드하면 Dynamic Media Classic에서 자산 ID가 &quot;shirt_front,&quot;&quot;shirt_flies&quot; 및 &quot;shirt_collar&quot;를 생성합니다. 이러한 옵션 중 하나를 사용하면 Dynamic Media Classic에서 이름이 고유한지 확인할 수 있습니다.

## 이미지 레이어로 템플릿 만들기

Dynamic Media Classic에서 레이어 PSD에서 템플릿을 자동으로 만들 수 있지만 수동으로 템플릿을 만드는 방법을 알고 있어야 합니다. 위에서 설명한 바와 같이, Dynamic Media Classic에서 만든 템플릿을 사용하지 않으려는 경우가 있습니다.

### 템플릿 기본 사항 UI

먼저 편집 인터페이스에 대해 잘 알아보겠습니다.

왼쪽 가운데에는 최종 템플릿의 미리 보기를 보여주는 작업 영역이 있습니다. 오른쪽에는 [레이어] 및 [레이어 속성] 패널이 있습니다. 이 영역들은 여러분이 가장 많은 일을 할 곳입니다.

![이미지](assets/basic-templates/basic-templates-5.jpg)

_템플릿 기본 사항 페이지를 참조하십시오._

- **미리 보기/작업 영역.** 여기가 메인 창입니다. 여기에서 마우스를 사용하여 레이어를 이동, 크기 조정 및 회전할 수 있습니다. 레이어 윤곽선은 파선으로 표시됩니다.
- **레이어.** 이는 Photoshop 레이어 패널과 유사합니다. 템플릿에 레이어를 추가하면 여기에 나타납니다. 레이어는 위에서 아래로 누적됩니다. [레이어] 패널의 위쪽 레이어는 목록의 맨 아래 다른 레이어 위에 표시됩니다.
- **레이어 속성.** 여기서는 숫자 컨트롤을 사용하여 레이어의 모든 속성을 조정할 수 있습니다. 먼저 레이어를 선택한 다음 해당 속성을 조정합니다.
- **** **CompositeURL.** UI 하단이 복합 URL 영역입니다. 이 내용은 자습서의 이 섹션에서 다루지 않지만, 여기서는 일련의 이미지 제공 URL 수정자로 구성된 템플릿을 확인할 수 있습니다. 이 영역은 편집할 수 있습니다. Image Server 명령에 익숙하다면 여기에서 템플릿을 수동으로 편집할 수 있습니다. 하지만 그것을 깨뜨릴 수도 있습니다. Photoshop과 마찬가지로 레이어 번호는 0부터 시작됩니다. Canvas는 레이어 0이고 직접 추가하는 첫 번째 레이어는 레이어 1입니다. 혼합 모드는 레이어의 픽셀이 그 아래에 있는 픽셀과 어떻게 혼합되는지를 결정합니다. 혼합 모드를 사용하여 다양한 특수 효과를 만들 수 있습니다.

#### 템플릿 기본 사항 편집기 사용

기본 템플릿을 시작하는 워크플로우 단계는 다음과 같습니다.

1. Dynamic Media Classic에서 **빌드 > 템플릿 기본 사항**&#x200B;으로 이동합니다. 아무것도 선택하지 않았거나, 이미지를 선택하여 시작하여 템플릿의 첫 번째 레이어가 됩니다.
2. [크기]를 선택하고 **확인**&#x200B;을 누릅니다. 이 크기는 Photoshop에서 디자인한 크기와 일치해야 합니다. 템플릿 편집기가 로드됩니다.
3. 1단계에서 이미지를 선택하지 않은 경우 왼쪽의 자산 패널에서 이미지를 검색하거나 찾아 작업 영역으로 끌어서 놓습니다.

   - 캔버스 크기에 맞게 이미지 크기가 자동으로 조정됩니다. 고해상도 이미지를 교체하려는 경우 일반적으로 큰(2000px) P-TIFF 이미지 중 하나를 자리 표시자로 사용합니다.
   - 템플릿의 맨 아래 레이어여야 하지만 나중에 레이어를 다시 정렬할 수 있습니다.

4. 작업 영역에서 직접 또는 [레이어 속성] 패널에서 설정을 조정하여 레이어의 크기를 조정하거나 위치를 변경합니다.
5. 필요에 따라 추가 이미지 레이어로 드래그합니다. 원하는 경우 레이어 효과를 추가합니다. 아래의 _레이어 효과 추가_ 항목을 참조하십시오.
6. **저장**&#x200B;을 클릭하고 위치를 선택한 다음 템플릿에 이름을 지정합니다. 미리 볼 수는 있지만 이 시점에서는 템플릿이 완전히 병합된 Photoshop 이미지와 동일하게 표시됩니다. 아직 변경할 수 없습니다.

### 레이어 효과 추가

이미지 서버는 몇 가지 프로그래밍 방식의 레이어 효과를 지원합니다. 레이어 컨텐츠의 모양을 변경하는 특수 효과입니다. 이 기능은 Photoshop의 레이어 효과와 유사하게 작동합니다. 레이어에 연결되지만 레이어와는 독립적으로 제어됩니다. 레이어 자체를 영구적으로 변경하지 않고 조정하거나 제거할 수 있습니다.

- **그림자**. x 및 y 픽셀 오프셋으로 배치된 레이어의 경계 바깥쪽에 그림자를 적용합니다.
- **내부 그림자**. x 및 y 픽셀 오프셋으로 배치된 레이어의 경계 내에 그림자를 적용합니다.
- **외부 글로우**. 레이어의 모든 가장자리 주위에 균일하게 광선 효과를 적용합니다.
- **내부 광선**. 레이어의 모든 가장자리 안에 균일하게 광선 효과를 적용합니다.

![이미지](assets/basic-templates/basic-templates-6.jpg)

_그림자 및 그림자 없는 레이어_

효과를 추가하려면 **효과 추가**&#x200B;를 클릭하고 메뉴에서 효과를 선택합니다. 일반 레이어처럼 [레이어] 패널에서 효과를 선택하고 [레이어 속성] 패널을 사용하여 해당 설정을 조정할 수 있습니다.

그림자 효과는 레이어에서 가로 또는 세로로 오프셋되지만 [광선] 효과는 모든 방향으로 균일하게 적용됩니다. [외부] 효과는 레이어의 불투명 부분 위에 영향을 주며 [외부] 효과는 투명 영역에만 영향을 줍니다.

[레이어 효과 추가](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/template-basics/creating-template.html#using-shadow-and-glow-effects-on-layers)에 대해 자세히 알아보십시오.

### 매개 변수 추가

레이어를 결합하고 저장하면 플랫 결과는 플랫 Photoshop 이미지와 다르지 않습니다. 템플릿을 특별하게 만드는 것은 각 레이어의 속성에 매개 변수를 추가하여 URL을 통해 동적으로 변경할 수 있는 기능입니다.

Dynamic Media Classic에서 매개 변수는 URL을 통해 조작할 수 있도록 템플릿 속성에 연결할 수 있는 변수입니다. 레이어에 매개 변수를 추가하면 Dynamic Media Classic이 달러 기호($)로 매개 변수의 이름을 접두사로 추가하여 URL에 해당 속성을 노출합니다. 예를 들어 &quot;size&quot; 라는 매개 변수를 만들어 레이어 크기를 변경하면 Dynamic Media Classic에서 매개 변수 이름을 $size로 변경합니다.

속성에 대한 매개 변수를 추가하지 않는 경우 해당 속성은 Dynamic Media Classic 데이터베이스에 숨겨져 있고 URL에 표시되지 않습니다.

![이미지](assets/basic-templates/parameters.png)

매개 변수가 없을 경우에는 일반적으로 URL이 훨씬 길어질 수 있습니다. 특히 다이내믹 텍스트도 사용하고 있습니다. 텍스트는 각 URL에 수십 개의 추가 문자를 추가합니다.

마지막으로, 초기 매개 변수 세트가 템플릿의 속성 기본값이 됩니다. 템플릿을 작성하고 매개 변수를 추가한 다음 매개 변수 없이 URL을 호출하는 경우 이미지 서버에서 템플릿에 저장한 모든 기본값을 사용하여 이미지를 생성합니다. 매개 변수는 속성을 변경하려는 경우에만 필요합니다. 속성을 변경할 필요가 없는 경우에는 매개 변수를 설정할 필요가 없습니다.

#### 매개 변수 만들기

매개 변수를 만드는 워크플로우입니다.

1. 매개 변수를 만들 레이어의 이름 옆에 있는 **매개 변수** 단추를 클릭합니다. 매개 변수 화면이 열립니다. 레이어의 각 속성과 해당 값이 나열됩니다.
1. 매개 변수로 만들 각 속성의 이름 옆에 있는 **On** 옵션을 선택합니다. 기본 매개 변수 이름이 나타납니다. 기본 상태에서 변경된 속성에 매개 변수만 추가할 수 있습니다.

   - 예를 들어 레이어를 추가하고 기본 xy 위치인 0,0에 유지하는 경우 Dynamic Media Classic에서는 **Position** 속성을 노출하지 않습니다. 수정하려면 하나 이상의 픽셀을 이동합니다. 이제 Dynamic Media Classic에서는 매개 변수화할 수 있는 속성으로 **Position**&#x200B;을 노출합니다.
   - 표시/숨기기 속성에 매개 변수를 추가하려면( 레이어를 켜거나 끄면) **Show** 또는 **레이어 숨기기** 아이콘을 클릭하여 레이어를 끄면 됩니다(필요한 경우 나중에 다시 켜서 해제할 수 있음). 이제 Dynamic Media Classic에 매개 변수화할 수 있는 **Hide** 속성이 표시됩니다.

1. 기본 매개 변수 이름을 URL에서 쉽게 식별할 수 있는 이름으로 변경합니다. 예를 들어, 이미지 위에 있는 배너 레이어를 변경하기 위해 매개 변수를 추가하려면 &quot;layer_2_src&quot;의 기본 이름을 &quot;banner&quot;로 변경하십시오.
1. **Close** 키를 눌러 Parameters 화면을 종료합니다.
1. **매개 변수** 단추를 클릭하고 매개 변수를 추가하고 이름을 바꾸면 다른 레이어에 대해 이 프로세스를 반복합니다.
1. 완료되면 변경 내용을 저장합니다.

>[!TIP]
>
>매개 변수의 이름을 의미 있는 이름으로 변경하고, 이름 지정 규칙을 개발하여 이러한 이름을 표준화합니다. 디자인 팀과 개발 팀 모두 이름 지정 규칙에 미리 동의해야 합니다.
>
>속성이 표시되지 않아 매개 변수를 추가할 수 없습니까? 이동, 크기 조정, 숨기기 등으로 레이어의 속성을 기본값에서 변경하면 됩니다. 이제 해당 속성이 노출되는 것이 보여야 합니다.

[템플릿 매개 변수](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/template-basics/creating-template-parameters.html)에 대해 자세히 알아보십시오.

## 텍스트 레이어로 템플릿 만들기

이제 텍스트 레이어를 포함하는 기본 템플릿을 만드는 방법을 알아봅니다.

### 다이내믹 텍스트 이해

이제 이미지 레이어를 사용하여 기본 템플릿을 만드는 방법을 알 수 있습니다. 많은 응용 프로그램의 경우 이 기능만 필요합니다. 이전 연습에서 보듯이 텍스트가 단순한 레이어(예: &quot;판매&quot; 및 &quot;신규&quot;)는 텍스트를 변경할 필요가 없으므로 래스터화하고 이미지로 처리할 수 있습니다.

그러나 필요한 경우 어떻게 해야 합니까?

- 25%라는 값이 변수와 함께 &quot;25% 해제&quot;라고 하는 레이블을 추가합니다
- 이미지 위에 제품 이름이 있는 텍스트 레이블을 추가합니다
- 템플릿이 표시되는 국가에 따라 레이어를 다른 언어로 현지화합니다

이 경우 텍스트 및/또는 서식을 제어하기 위해 매개 변수와 함께 동적 텍스트 레이어를 추가하려고 합니다.

텍스트를 만들려면 일부 글꼴을 업로드해야 합니다. 그렇지 않으면 Dynamic Media Classic이 기본적으로 Arial로 설정됩니다. 글꼴은 이미지 서버에도 게시해야 합니다. 그렇지 않으면 해당 글꼴을 사용하는 텍스트를 렌더링하려고 할 때 오류가 발생합니다.

### RTF 및 텍스트 매개변수

템플릿 기본 사항 도구를 사용하여 텍스트에 변수를 추가하려면 텍스트가 렌더링되는 방식을 이해해야 합니다. 이미지 서버는 Photoshop과 Illustrator에서 사용하는 동일한 엔진인 Adobe 텍스트 엔진을 사용하여 텍스트를 생성하고 최종 이미지의 레이어로 합성합니다. 엔진과 통신하기 위해 이미지 서버는 리치 텍스트 형식 또는 RTF를 사용합니다.

RTF는 Microsoft에서 문서 서식을 지정하기 위해 개발한 파일 형식 사양입니다. 대부분의 워드 프로세싱 및 이메일 소프트웨어에서 사용하는 표준 마크업 언어입니다. URL &amp;text=\b1 Hello에 입력한 경우 \b1은 텍스트를 굵게 만들기 위한 RTF 명령이므로 이미지 서버에서 &quot;Hello&quot;라는 단어가 굵은 형식으로 포함된 이미지를 생성합니다.

좋은 소식은 Dynamic Media Classic에서 RTF를 생성한다는 것입니다. 서식 파일에 텍스트를 입력하고 서식을 추가할 때마다, Dynamic Media Classic에서는 서식 파일에 RTF 코드를 자동으로 기록합니다. 이 문제를 언급하는 이유는 매개 변수를 RTF 자체에 직접 추가할 예정이기 때문입니다. 따라서 이 매개 변수를 잘 알고 있어야 합니다.

#### 텍스트 레이어 만들기

Dynamic Media Classic에서 다음 두 가지 방법으로 템플릿에 텍스트 레이어를 만들 수 있습니다.

1. Dynamic Media Classic의 텍스트 도구. 이 방법은 아래에 설명되어 있습니다. 템플릿 기본 사항 편집기에는 텍스트 상자를 만들고 텍스트를 입력하고 텍스트 서식을 지정할 수 있는 도구가 있습니다. Dynamic Media Classic은 필요에 따라 RTF를 생성하여 별도의 레이어에 배치합니다.
2. 텍스트 추출(업로드 시). 다른 방법은 Photoshop에서 텍스트 레이어를 만들어 일반 텍스트 레이어로 저장하는 것입니다(이미지 레이어로 래스터화하는 대신). 그런 다음 파일을 Dynamic Media Classic에 업로드하고 **텍스트 추출** 옵션을 사용합니다. Dynamic Media Classic에서는 RTF 명령을 사용하여 각 Photoshop 텍스트 레이어를 이미지 제공 텍스트 레이어로 변환합니다. 이 방법을 사용하는 경우 먼저 글꼴을 Dynamic Media Classic에 업로드하십시오. 그렇지 않으면 업로드 시 Dynamic Media Classic이 기본 글꼴로 대체되며, 올바른 글꼴을 다시 대체하는 쉬운 방법은 없습니다.

### 텍스트 편집기

텍스트 편집기를 사용하여 텍스트를 입력합니다. 텍스트 편집기는 Photoshop 또는 Illustrator과 유사한 서식 컨트롤을 사용하여 텍스트를 입력하고 서식을 지정할 수 있는 WYSIWYG 인터페이스입니다.

![이미지](assets/basic-templates/basic-templates-9.jpg)

_템플릿 기본 사항 텍스트 편집기._

대부분의 작업은 **미리 보기** 탭에서 수행합니다. 이 탭에서는 텍스트를 입력하고 템플릿에 표시될 내용을 볼 수 있습니다. 필요한 경우 RTF를 수동으로 편집하는 데 사용되는 **원본** 탭도 있습니다.

일반 워크플로우는 **미리 보기** 탭을 사용하여 일부 텍스트를 입력하는 것입니다.

텍스트를 선택하고 맨 위에 있는 컨트롤을 사용하여 글꼴 색, 글꼴 크기 또는 맞춤과 같은 서식을 선택합니다. 텍스트 스타일이 원하는 방식으로 지정된 후 **적용**&#x200B;을 클릭하여 작업 영역 미리 보기에서 업데이트를 확인합니다. 그런 다음 텍스트 편집기를 닫고 템플릿 기본 창으로 돌아갑니다.

#### 텍스트 편집기 사용

다음은 템플릿 기본 사항 빌드 페이지 내에 텍스트를 추가하는 워크플로우 단계입니다.

1. 빌드 페이지의 맨 위에 있는 **텍스트** 도구 단추를 클릭합니다.
2. 텍스트를 표시할 텍스트 상자를 드래그합니다. 텍스트 편집기 창이 모달 창에 열립니다. 배경에는 템플릿이 표시되지만 텍스트 편집을 완료할 때까지 편집할 수 없습니다.
3. 템플릿을 처음 로드할 때 표시할 샘플 텍스트를 입력합니다. 예를 들어 개인화된 이메일 이미지용 텍스트 상자를 만드는 경우 텍스트에 &quot;Hi Name&quot;이 표시됩니다. 이제 저축할 때입니다!&quot; 나중에 이름 을 URL에서 보내는 값으로 대체하기 위해 텍스트 매개 변수를 추가합니다. **적용**&#x200B;을 클릭하기 전까지는 창 아래의 템플릿에 텍스트가 표시되지 않습니다.
4. 텍스트 서식을 지정하려면 마우스를 사용하여 텍스트를 선택하고 UI에서 서식 컨트롤을 선택합니다.

   - 다양한 서식 옵션이 있습니다. 가장 일반적으로 글꼴(얼굴), 글꼴 크기 및 글꼴 색뿐만 아니라 왼쪽/가운데/오른쪽 맞춤도 있습니다.
   - 텍스트를 먼저 선택하는 것을 잊지 마십시오. 그렇지 않으면 서식을 적용할 수 없습니다.
   - 다른 글꼴을 선택하려면 텍스트를 선택하고 [글꼴] 메뉴를 엽니다. 편집기에 Dynamic Media Classic에 업로드된 모든 글꼴의 목록이 표시됩니다. 글꼴이 컴퓨터에 설치되어 있으면 검정색으로 표시됩니다. 컴퓨터에 설치되어 있지 않으면 빨간색으로 표시됩니다. 그러나 **적용**&#x200B;을 클릭해도 여전히 미리 보기 창에서 렌더링됩니다. Dynamic Media Classic을 사용하는 모든 사용자가 글꼴을 사용할 수 있도록 하려면 글꼴을 Dynamic Media Classic에 업로드하기만 하면 됩니다. 게시하면 이미지 서버에서 해당 글꼴을 사용하여 텍스트를 생성합니다. 글꼴은 이미지의 일부이므로 사용자가 직접 만든 텍스트를 볼 필요가 없습니다.
   - Photoshop 및 Illustrator과 달리 이미지 서버는 텍스트 상자에서 텍스트를 세로로 정렬할 수 있습니다. 기본값은 위쪽 정렬입니다. 이를 변경하려면 텍스트를 선택하고 **세로 정렬** 메뉴에서 **중간** 또는 **아래쪽**&#x200B;을 선택합니다.
   - 텍스트를 상자에 너무 크게 하거나 텍스트 상자가 너무 작으면 전체 또는 일부가 잘려서 사라집니다. 글꼴 크기를 줄이거나 상자를 크게 만듭니다.

5. 작업 영역 창에서 변경 내용이 적용되도록 하려면 **적용**&#x200B;을 클릭하십시오. **적용**&#x200B;을 클릭해야 합니다. 그렇지 않으면 편집 내용이 손실됩니다.
6. 완료되면 **닫기**&#x200B;를 클릭하십시오. 편집 모드로 돌아가려면 텍스트 레이어를 두 번 클릭하여 텍스트 편집기를 다시 엽니다.

시스템에 글꼴이 로컬에 설치된 경우 텍스트 편집기에서 글꼴의 크기를 정확히 미리 봅니다.

### 텍스트 레이어에 매개 변수 추가 정보

이제 레이어 매개 변수에서 그랬던 것처럼 텍스트 매개 변수를 추가하는 비슷한 프로세스를 따릅니다. 텍스트 레이어는 크기, 위치 등의 레이어 매개 변수를 사용할 수도 있습니다. 그러나 RTF의 모든 측면을 제어할 수 있는 추가 매개변수를 사용할 수 있습니다.

레이어 매개 변수와 달리 변경할 값을 선택하고 전체 속성에 매개 변수를 추가하는 대신 매개 변수만 추가합니다.

![이미지](assets/basic-templates/basic-templates-10.jpg)

샘플 RTF:

![이미지](assets/basic-templates/sample-rtf.png)

RTF를 검사할 때 각 설정을 변경할 위치를 파악해야 합니다. 위의 RTF에서는 일부 서식이 적절할 수 있으며 서식이 어디에서 오는지 볼 수 있습니다.

여러분은 초콜릿 민트 샌달이라는 말을 볼 수 있습니다. 그것은 텍스트 그 자체입니다.

- Poor Richard 글꼴에 대한 참조가 있습니다. 여기서 글꼴이 선택됩니다.
- RGB 값을 볼 수 있습니다. \red56\green53\blue4 — 텍스트 색상입니다.
- 글꼴 크기가 20이지만 20이라는 숫자가 표시되지 않습니다. 그러나 \fs40 명령이 표시됩니다. 일부 이상한 이유로 RTF는 글꼴을 반점으로 측정합니다. 따라서 \fs40은 글꼴 크기입니다!

매개 변수를 작성할 수 있는 충분한 정보가 있지만 이미지 제공 문서에 모든 RTF 명령에 대한 전체 참조가 있습니다. [이미지 제공 설명서](https://docs.adobe.com/content/help/en/dynamic-media-developer-resources/image-serving-api/image-serving-api/http-protocol-reference/text-formatting/c-text-formatting.html#concept-0d3136db7f6f49668274541cd4b6364c)를 방문하십시오.

#### 텍스트 레이어에 매개 변수 추가

텍스트 레이어에 매개 변수를 추가하는 단계는 다음과 같습니다.

1. 매개 변수를 만들 텍스트 레이어의 이름 옆에 있는 **매개 변수** 단추(&quot;P&quot;)를 클릭합니다. 매개 변수 화면이 열립니다. **공통** 탭에는 레이어의 각 속성과 해당 값이 나열됩니다. 여기서 일반 레이어 매개 변수를 추가할 수 있습니다.
1. **텍스트** 탭을 클릭합니다. 여기 맨 위에 RTF가 있습니다. 추가하는 매개 변수는 그 아래에 있습니다.
1. 매개 변수를 추가하려면 먼저 변경할 값을 강조 표시하고 **매개 변수 추가** 단추를 클릭합니다. 전체 명령 자체가 아니라 명령에 대한 값만 선택해야 합니다. 예를 들어, 위의 샘플 RTF에서 글꼴 이름에 대한 매개변수를 설정하려면 &quot;Fluid Richard&quot;를 강조 표시하고 매개 변수만 추가하되 &quot;\f0&quot;은 추가하지 않습니다. **매개 변수 추가** 를 클릭하면 아래 목록에 표시되고 매개변수 값은 여전히 선택된 동안 RTF에서 빨간색으로 표시됩니다. 매개 변수를 제거해야 하는 경우 **On** 옆의 확인란을 클릭하여 해당 매개 변수를 끄면 해당 매개 변수가 사라집니다.
1. 를 클릭하여 매개 변수의 이름을 더 의미 있는 이름으로 변경합니다.
1. 그러면 매개변수 이름이 있는 RTF가 녹색으로 강조 표시되고 매개변수 이름과 값이 아래에 나열됩니다.
1. **닫기**&#x200B;를 클릭하여 매개 변수 화면을 종료합니다. 그런 다음 **Save** 를 눌러 템플릿을 저장합니다. 편집을 마친 경우 **Close** 키를 눌러 [템플릿 기본 사항] 페이지를 종료합니다.
1. **미리 보기**&#x200B;를 클릭하여 Dynamic Media Classic에서 템플릿을 테스트합니다. 텍스트 매개 변수를 테스트하려면 미리 보기 창에 새 텍스트 또는 새 값을 입력합니다. 글꼴을 변경하려면 글꼴의 정확한 RTF 이름을 입력해야 합니다.

>[!TIP]
>
>텍스트 색상에 매개 변수를 추가하려면 빨간색, 녹색 및 파란색에 대한 매개 변수를 별도로 추가하십시오. 예를 들어 RTF가 `\red56\green53\blue46`이면 56, 53 및 46에 대해 별도의 빨간색, 녹색 및 파란색 매개변수를 추가합니다. URL에서 세 가지 모두 를 호출하여 색상을 변경합니다. `&$red=56&$green=53&$blue=46`

[동적 텍스트 매개 변수를 만드는 방법을 알아봅니다](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/template-basics/creating-template-parameters.html#creating-dynamic-text-parameters).

## 템플릿 URL 게시 및 만들기

### 이미지 사전 설정 만들기

템플릿에 대한 사전 설정을 만드는 것은 필수 단계가 아닙니다. 템플릿을 항상 1:1 크기로 호출하고 템플릿에 맞게 크기가 조정된 큰 이미지 레이어에 선명하게 하기를 추가하는 것이 가장 좋습니다. 사전 설정 없이 이미지를 호출하는 경우 이미지 서버가 이미지 크기를 기본 크기(약 400픽셀)로 임의로 조정할 수 있으며 기본 선명하게 하기는 적용되지 않습니다.

템플릿에 대한 이미지 사전 설정에 대해서는 특별한 것이 없습니다. 같은 크기의 정적 이미지에 대한 사전 설정이 이미 있는 경우 대신 사용할 수 있습니다.

### 게시

이미지 서버에 실시간으로 푸시된 변경 사항을 보려면 게시를 실행해야 합니다. 게시해야 하는 사항을 염두에 두십시오. 다양한 이미지 자산 레이어, 동적 텍스트에 대한 글꼴 및 템플릿 자체. 이미지 세트 및 스핀 세트와 같은 다른 Dynamic Media Classic 리치 미디어 자산과 유사한 기본 템플릿은 인공 구조입니다. 일련의 이미지 제공 명령을 사용하여 이미지와 글꼴을 참조하는 데이터베이스의 라인 항목입니다. 따라서 템플릿을 게시하면 이미지 서버에서 데이터를 업데이트하기만 하면 됩니다.

[템플릿 게시](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/template-basics/publishing-templates.html)에 대해 자세히 알아보십시오.

### 템플릿 URL 구성

기본 템플릿에는 앞에서 설명한 일반적인 이미지 호출과 동일한 필수 URL 구문이 있습니다. 템플릿에는 일반적으로 값이 있는 매개 변수와 같이 앰퍼샌드(&amp;)로 구분된 명령 등 더 많은 수정자가 있습니다. 하지만 주 차이점은 정적 이미지에 호출하는 대신 템플릿을 기본 이미지로 호출하는 것입니다.

![이미지](assets/basic-templates/basic-templates-11.jpg)

사전 설정된 이름의 양쪽에 달러 기호($)가 있는 이미지 사전 설정과 달리 매개 변수는 시작 부분에 단일 달러 기호가 있습니다. 그 달러 표지판의 배치가 중요합니다.

**올바른:**

`$text=46-inch LCD HDTV`

**잘못된:**

`$text$=46-inch LCD HDTV`

`$text=46-inch LCD HDTV$`

`text=46-inch LCD HDTV`

앞에서 언급한 바와 같이, 매개 변수는 템플릿을 변경하는 데 사용됩니다. 매개 변수 없이 템플릿을 호출하면 템플릿 기본 작성 도구에 설명된 대로 기본 설정으로 돌아갑니다. 속성을 변경할 필요가 없는 경우에는 매개 변수를 설정할 필요가 없습니다.
