# 자르기, 조정된 이미지 및 확대/축소 Target {#crop-adjusted-zoom-targets}

Dynamic Media Classic 마스터 이미지 개념의 주요 장점 중 하나는 많은 용도로 이미지 자산을 재활용할 수 있다는 것입니다. 일반적으로 세부 사항이나 색상 견본을 표시하려면 각 이미지의 잘린 버전을 별도로 만들어야 합니다. Dynamic Media Classic을 사용하는 경우 단일 마스터에서 동일한 작업을 수행하고 이러한 잘린 버전을 새 물리적 파일 또는 저장 공간 없이 가상 변형 파일로 저장할 수 있습니다.

자습서의 이 섹션 끝 부분에는 다음 방법을 알 수 있습니다.

- Dynamic Media Classic에서 이미지를 자르고 새 마스터 파일 또는 가상 이미지로 저장합니다. [추가 정보](https://docs.adobe.com/help/en/dynamic-media-classic/using/master-files/cropping-image.html).
- 가상 조정된 이미지를 저장하고 마스터 자산 대신 사용합니다. [추가 정보](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/master-files/adjusting-image.html).
- 이미지에 확대/축소 Target을 만들어 밝은 영역을 표시합니다. [추가 정보](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/zoom/creating-zoom-targets-guided-zoom.html).

## 자르기

Dynamic Media Classic에는 자르기 툴을 비롯하여 UI에서 편리하게 사용할 수 있는 몇 가지 이미지 편집 도구가 있습니다. 여러 가지 이유로 Dynamic Media Classic에서 마스터 이미지를 잘라낼 수 있습니다. 예:

- 원본 파일에 액세스할 수 없습니다. 이미지를 다른 자르기 또는 종횡비로 표시하려고 하지만 컴퓨터에 원본 파일이 없거나 집에서 작업하고 있습니다. 이 경우 Dynamic Media Classic으로 이동하여 이미지를 찾아 자르고 저장하거나 새 버전으로 저장할 수 있습니다.
- 불필요한 공백을 제거하려면 하얀 공간이 너무 많은 사진이 찍혔기 때문에 제품이 작아 보입니다. 축소판 이미지가 캔버스를 최대한 채우도록 할 수 있습니다.
- 조정된 이미지를 만들려면 디스크 공간이 없는 이미지의 가상 사본을 만듭니다. 일부 회사에는 동일한 이미지의 사본을 별도로 보관해야 하지만 이름이 다른 비즈니스 규칙이 있습니다. 또는 동일한 이미지의 잘리고 잘리지 않는 버전을 원할 수도 있습니다.
- 소스 이미지에서 새 이미지를 만들려면 예를 들어 색상 견본 또는 기본 이미지의 세부 사항을 만들 수 있습니다. Adobe Photoshop에서 이 작업을 수행하고 별도로 업로드하거나 Dynamic Media Classic에서 자르기 도구를 사용할 수 있습니다.

>[!NOTE]
>
>다음 자르기 토론에 있는 모든 URL은 실례만을 위한 것입니다.라이브 링크가 아닙니다.

### 자르기 도구 사용

에셋의 세부 정보 페이지에서 또는 **편집** 단추를 클릭하여 Dynamic Media Classic의 자르기 도구에 액세스할 수 있습니다. 이 도구를 사용하여 다음 두 가지 방법으로 자를 수 있습니다.

- 자르기 창의 핸들을 드래그하거나 [크기] 상자에 값을 입력하는 기본 자르기 모드입니다. [수동으로 자르기](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/master-files/cropping-image.html#select-an-area-to-crop)하는 방법을 알아봅니다.
- 트리밍. 이미지와 일치하지 않는 픽셀 수를 계산하여 이미지 주위의 추가 공백을 제거하려면 이 옵션을 사용합니다. [트리밍하여 자르기](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/master-files/cropping-image.html#crop-to-remove-white-space-around-an-image)하는 방법을 알아봅니다.

### _수동 자르기_

수동으로 잘린 버전을 저장하면 이미지가 영구적으로 잘려 보입니다.Dynamic Media Classic에서는 이미지를 자르는 데 내부 URL 수정자를 추가하여 픽셀을 실제로 숨기고 있습니다. 게시하면 이미지가 잘리는 것처럼 모든 사람에게 표시되지만 자르기 편집기로 돌아가 나중에 자르기를 제거할 수 있습니다.

그런 다음 새 기본 이미지로 저장할지 또는 마스터의 추가 보기로 저장할지를 선택할 수 있습니다. 새 마스터는 저장 공간을 차지하는 새로운 물리적 파일(예: TIFF 또는 JPEG)입니다. 추가 보기는 서버 공간을 차지하지 않는 가상 이미지입니다. 마스터를 덮어쓰고 자르기를 영구적으로 만들기 때문에 [원본 바꾸기]를 선택하지 않는 것이 좋습니다. 새 마스터 또는 추가 보기로 저장하는 경우 새 자산 ID를 선택해야 합니다. 다른 자산 ID와 마찬가지로 Dynamic Media Classic에서 고유한 이름이어야 합니다.

### _자르기 트리밍_

이미지의 주 제목 주위에 공백(추가 캔버스)이 너무 많은 이미지를 업로드하면 크기를 조정할 때 웹에서 이미지가 훨씬 작아집니다. 특히 150픽셀 이하의 축소판 이미지에 대해서는 이러한 문제가 발생합니다. 사진 주위에는 추가 공간이 모두 손실될 수 있습니다.

동일한 이미지의 두 버전을 비교합니다.

![이미지](assets/crop-adjusted-zoom-targets/trim-cropping.jpg)

오른쪽에 있는 이미지는 제품 주변의 추가 공간을 제거하여 훨씬 더 눈에 띄게 됩니다. 트리밍은 자르기 툴을 사용하여 한 번에 하나의 이미지를 수행하거나 업로드하는 동안 일괄 처리로 실행할 수 있습니다. 모든 이미지를 주제의 경계로 일관되게 잘리게 하려는 경우 일괄 처리 방식으로 실행하는 것이 좋습니다. 이미지를 둘러싼 사각형인 테두리 상자로 잘라냅니다.

>[!NOTE]
>
>트리밍은 이미지 주위에 투명도를 만들지 않습니다. 이를 위해 이미지에 클리핑 패스를 포함시키고 **클립 패스에서 마스크 만들기** 업로드 옵션을 사용해야 합니다.
>
>또한 **저장** 옵션을 사용할 때 이미지를 잘라낸 후 이미지를 원래 상태로 복원하려면 [자르기 편집기] 화면에 이미지를 표시하고 **재설정** 단추를 선택합니다.

### _업로드 시 자르기_

앞에서 언급했듯이 이미지를 업로드할 때 이미지를 자를 수도 있습니다. 업로드 시 트림 자르기를 사용하려면 **작업 옵션** 단추를 클릭하고 [자르기 옵션] 아래에서 **트림**&#x200B;을 선택합니다.

Dynamic Media Classic에서는 다음 업로드를 위해 이 옵션을 기억합니다. 이 업로드를 위해 이미지를 자르려면 모든 업로드에 대해 이미지를 잘리지 않도록 해야 합니다. 다른 옵션은 특별히 예약된 FTP 업로드 작업을 설정하고 자르기 옵션을 여기에 배치하는 것입니다. 이렇게 하면 이미지를 자르는 데 필요한 경우에만 작업을 실행할 수 있습니다.

>[!IMPORTANT]
>
>업로드할 자르기를 설정하면 Dynamic Media Classic에서 다음에 해당 설정을 기억하기 위한 쿠키를 저장합니다. 가장 좋은 방법은 다음 업로드 전에 **회사 기본값으로 재설정** 단추를 클릭하여 마지막 업로드에서 남은 자르기 옵션을 지우는 것입니다.그렇지 않은 경우 다음 이미지 일괄 자르기를 실수로 할 수 있습니다.

### URL로 자르기

Dynamic Media Classic에서는 잘 보이지 않지만 URL을 통해서도 자르거나 이미지 사전 설정에 자르기를 추가할 수도 있습니다.

자르기 도구를 사용할 때마다 맨 아래에 있는 필드에 URL 값이 표시됩니다. 이러한 값을 가져와 URL 수정자로 이미지에 직접 적용할 수 있습니다.

![이미지](assets/crop-adjusted-zoom-targets/cropping-by-url.png)
_자르기 편집기 아래쪽에 있는 자르기 명령 수정자_

![이미지](assets/crop-adjusted-zoom-targets/uncropped-cropped.png)

트리밍으로 자르기를 사용하는 경우 이미지 기준으로 크기를 계산해야 하므로 URL을 통해 크기를 자동화하지 못합니다. 트림 자르기는 업로드 시 또는 한 번에 하나의 이미지를 적용하여 실행할 수 있습니다.

### _이미지 사전 설정에서 자르기_

이미지 사전 설정에는 이미지 제공 명령을 추가할 수 있는 필드가 있습니다. 이미지 사전 설정에 위와 동일한 자르기를 추가하려면 사전 설정을 편집하고 붙여넣거나 URL 수정자 필드에 값을 입력한 다음 저장하고 게시합니다.

![이미지](assets/crop-adjusted-zoom-targets/cropping-in-image-preset.jpg)
_이미지 사전 설정의 URL 수정자에 자르기 명령(또는 명령)을 추가합니다._

자르기는 이제 해당 이미지 사전 설정의 일부가 되며 사용할 때마다 자동으로 적용됩니다. 물론 이 방법은 동일한 자르기 양이 필요한 모든 이미지에 따라 달라집니다. 이미지가 동일한 방식으로 촬영되지 않는 경우 이 방법이 작동하지 않을 수 있습니다.

## 조정된 이미지

자르기 도구를 사용하는 경우 **의 추가 보기로 기본**&#x200B;저장을 선택할 수 있습니다. 저장되면 새로운 종류의 Dynamic Media Classic 자산(조정된 이미지)이 만들어집니다. 파생 이미지라고도 하는 조정된 이미지는 가상 이미지입니다. 그것은 실제로 전혀 이미지가 아닙니다.실제 마스터 이미지에 대한 데이터베이스 참조(별칭 또는 바로 가기)입니다.

### 실제 이미지가 위로`?`

마스터가 무엇이고, 조정된 이미지가 어느 것인지 알 수 있습니까?

![이미지](assets/crop-adjusted-zoom-targets/real-image-stand-up.png)

Dynamic Media Classic을 보고 SBR_MAIN2에 대한 &quot;조정된 이미지&quot;의 자산 유형을 보지 않고는 알 수 없습니다.

조정된 이미지는 데이터베이스의 라인 항목으로만 존재하므로 디스크 공간을 사용하지 않습니다. 이것은 또한 원래 자산에 영구적으로 연결되어 있습니다;원본이 삭제되면 조정된 이미지도 삭제됩니다. 이미지의 전체, 잘리지 않은 이미지 또는 이미지의 일부(자르기)로 구성될 수 있습니다.

![이미지](assets/crop-adjusted-zoom-targets/adjusted-image.png)

일반적으로 자르기 도구를 사용하여 조정된 이미지를 만듭니다.그러나 조정 및 선명 효과 툴인 다른 이미지 편집기로도 만들 수 있습니다.

조정된 이미지에는 고유한 자산 ID가 필요합니다. 게시되면(다른 자산과 같이 게시해야 함) 다른 이미지로 작동하며 자산 ID로 URL에 호출됩니다. 세부 사항 페이지에서 **기본 제공 및 파생** 탭 아래에서 마스터 이미지와 연결된 조정된 이미지를 볼 수 있습니다.

![이미지](assets/crop-adjusted-zoom-targets/derivatives.jpg)
_마스터 이미지에 대한 조정된 보기 ASIAN_BR_MAIN_

## 확대/축소 Target

확대/축소 Target은 이미지의 **편집** 메뉴 및 **세부 사항** 페이지에서도 찾을 수 있습니다. 확대/축소 이미지의 특정 머천다이징 기능을 강조하도록 &quot;핫 스팟&quot;을 설정할 수 있습니다. 큰 마스터를 잘라내어 별도의 이미지를 만드는 대신, 확대/축소 뷰어는 만드는 짧은 레이블과 함께 이미지 위에 세부 사항을 표시할 수 있습니다.

![이미지](assets/crop-adjusted-zoom-targets/arm-with-watch.jpg)

확대/축소 Target은 본질적으로 머천다이징 기능이며 제품의 판매 지점에 대한 지식이 필요하기 때문에 일반적으로 머천다이징 또는 제품 팀의 한 사람이 회사의 판매 지점을 만듭니다.

이 프로세스는 매우 간단합니다. 기능을 클릭하고 설명형 이름을 지정한 다음 저장합니다. Target이 유사한 경우 한 이미지에서 다른 이미지로 복사할 수 있지만 프로세스는 수동입니다. Dynamic Media Classic에서는 각 이미지가 서로 다르며 기능이 다르므로 확대/축소 Target 만들기를 자동화할 수 없습니다.

확대/축소 Target 사용 여부를 결정하는 또 다른 요소는 뷰어 선택입니다. 일부 뷰어 유형에서는 확대/축소 Target을 표시할 수 없습니다(예: 플라이아웃 뷰어는 지원하지 않음).

[확대/축소 Target 만들기](https://docs.adobe.com/content/help/en/dynamic-media-classic/using/zoom/creating-zoom-targets-guided-zoom.html#creating-and-editing-zoom-targets) 방법을 알아봅니다.

![이미지](assets/crop-adjusted-zoom-targets/zoom-targets.jpg)

### 확대/축소 Target 도구 사용

다음은 Dynamic Media Classic에서 대상을 만드는 워크플로우입니다.

1. 이미지를 탐색하고 **편집** 단추를 클릭한 다음 **Target 확대/축소**&#x200B;를 선택합니다.
2. 확대/축소 Target 편집기가 로드됩니다. 이미지의 중앙에, 맨 위에 있는 버튼, 오른쪽에 빈 대상 패널이 표시됩니다. 왼쪽 아래에 뷰어 사전 설정이 선택되어 있습니다. 기본값은 &quot;확대/축소1-안내&quot;입니다.
3. 마우스로 빨간색 상자를 이동하고 클릭하여 새 대상을 만듭니다.

   - 빨간색 상자는 대상 영역입니다. 사용자가 해당 타겟을 클릭하면 상자 안의 영역으로 확대됩니다.
   - 대상 크기는 뷰어 사전 설정 내의 보기 크기에 의해 결정됩니다. 기본 확대/축소 이미지의 크기를 결정합니다. 아래의 _보기 크기_ 설정을 참조하십시오.

4. 방금 만든 타겟이 파란색으로 바뀝니다. 오른쪽에는 기본 이름 &quot;target-0&quot;뿐만 아니라 해당 대상의 축소판 버전이 표시됩니다.
5. 대상의 이름을 바꾸려면 해당 축소판을 클릭하고 새 **이름**&#x200B;을 입력한 다음 **Enter** 또는 **Tab**&#x200B;을 클릭합니다. 여기를 클릭만 하면 이름이 저장되지 않습니다.
6. 대상을 선택하면 상자 주위에 녹색 점선이 표시되고 크기를 조정하고 이동할 수 있습니다. 모서리를 드래그하여 크기를 조정하거나 대상 상자를 드래그하여 이동합니다.

   - 기본 사용자 정의 확대/축소 뷰어 내에 이미지가 로드됩니다. 뷰어 사전 설정이 확대/축소 Target을 지원하는지 확인하십시오. 일반적으로 &quot;-안내&quot;라는 단어가 있는 모든 표준 사전 설정은 확대/축소 Target에서 사용하도록 설계되었습니다. 대상을 사용하려면 대상 축소판(또는 핫스팟 아이콘) 위로 마우스를 가져가 레이블을 확인한 다음 클릭하여 해당 기능을 확대/축소하는 뷰어를 확인합니다.
   - Dynamic Media Classic에서 수행하는 다른 모든 작업과 마찬가지로, 확대/축소 Target이 웹에서 라이브되도록 게시해야 합니다. 타겟을 지원하는 뷰어를 이미 사용하고 있는 경우 즉시(캐시가 지워진 후) 표시됩니다. 그러나 확대/축소 Target 사용 뷰어를 사용하지 않을 경우 숨겨진 상태로 유지됩니다.

      ![이미지](assets/crop-adjusted-zoom-targets/zoom-target-green-box.jpg)

7. 또한 대상을 제거해야 하는 경우 축소판을 클릭하여 선택하고 **Target 삭제** 단추를 누르거나 키보드의 DELETE 키를 누릅니다.
8. 추가 후 계속 클릭하여 새 타겟을 추가하거나, 이름을 변경하고, 크기를 조정합니다.
9. 완료되면 **저장** 단추를 클릭한 다음 **미리 보기**&#x200B;를 클릭합니다.

### 확대/축소 뷰어 사전 설정에서 보기 크기 설정

확대/축소 Target의 크기가 어디에서 오는지 잠시 살펴보겠습니다. 확대/축소 뷰어의 뷰어 사전 설정 안에는 보기 크기인 설정이 있습니다. 보기 크기는 뷰어 내의 확대/축소 이미지 크기입니다. UI 구성 요소 및 아트웍을 포함하여 뷰어의 총 크기인 스테이지 크기와 다릅니다.

새 대상을 만들 때 보기 크기에서 해당 크기와 종횡비를 파생합니다. 예를 들어 보기 크기가 200 x 200이면 최대 확대/축소 영역이 200픽셀인 정사각형 대상만 만들 수 있습니다. 타겟은 200픽셀보다 클 수 있지만 항상 정사각형입니다. 그러나 확대/축소 뷰어 내의 이미지는 200픽셀뿐입니다. 확대/축소 대상의 크기는 뷰어 크기와 직접적으로 관련이 있습니다. 따라서 타겟을 설정하기 전에 먼저 뷰어 디자인을 결정할 수 있습니다.

그러나 기본 보기 이미지의 크기는 동적이며 스테이지 크기에 따라 자동으로 파생되므로 기본적으로 보기 크기는 비어 있습니다(0 x 0으로 설정). 사전 설정에 보기 크기를 명시적으로 설정하지 않으면 [확대/축소 Target] 도구가 타겟을 만드는 크기를 알 수 없습니다.

[확대/축소 Target] 도구를 로드하면 사전 설정 이름 옆에 보기 크기가 표시됩니다. 내장된 확대/축소1 안내 사전 설정과 사용자 정의 ZT_AUTHORING 사전 설정 간의 보기 크기를 비교합니다.

![이미지](assets/crop-adjusted-zoom-targets/view-size.jpg)

내장 사전 설정의 크기가 900 x 550인 것을 볼 수 있습니다. 즉, 대상이 크기보다 더 작아질 수 없습니다. 크기가 너무 큽니다. 2000픽셀 이미지가 있는 경우 가로 900픽셀 이상의 기능만 호출하면 됩니다. 사용자가 수동으로 추가로 확대/축소할 수 있지만 더 이상 자세히 볼 수 없습니다. 보기 크기를 350 x 350으로 설정하면 타겟이 매우 가깝게 확대하거나 크기를 크게 조정할 수 있습니다. 뷰어에서 확대/축소 이미지를 확대하려면 350픽셀로 잠겨 있으므로 새 사전 설정을 만들어야 합니다.

### 확대/축소 Target을 지원하는 뷰어 사전 설정 만들기 또는 편집

보기 크기를 설정하려면 확대/축소 Target을 지원하는 뷰어 사전 설정을 만들거나 편집합니다.

1. 뷰어 사전 설정에서 **확대/축소 설정** 옵션으로 이동합니다.
2. 폭과 높이를 설정합니다.
3. 사전 설정을 저장하고 닫습니다. 라이브 사이트에서 해당 사전 설정을 사용하려면 나중에 게시도 해야 합니다.
4. [Target 확대/축소] 도구로 이동하여 왼쪽 하단에 편집한 사전 설정을 선택합니다. 타겟에 반영된 새 보기 크기가 즉시 표시됩니다.
